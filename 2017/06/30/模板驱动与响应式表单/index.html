    • 前言-两种风格

    Angular4 提供两种类型的表单处理机制，模板驱动(Template Driven)和模型驱动(Model Driven)，而模型驱动表单又叫响应式表单(Reactive Froms)。模板驱动表单与Angular1.x 其实是一个套路。

    响应式表单是同步的,模板驱动表单是异步的。模板驱动型表单是将更多的工作交给了模板来处理。对于不太复杂的表单，模块驱动型表单可能更合适。但是其可测试能力是比不过响应式表单的，并且使用响应式我们将具备对表单的完全控制权。而配合 Rx.js 的响应式表单将发挥更加强大的威力。

    模板驱动表单

    我们先来看看模板驱动型表单。

    下面是模板型驱动表单中常用到的指令:

    NgFrom

    NgForm 指令: Angular 会自动在 form 表单上创建并附加该指令，它代表该表单的整体。 它会控制那些带有ngModel指令和name属性的元素，监听他们的属性。而其本身的 valid 属性则只有当所有的表单控件都有效的时候才会返回 true。

    我们导入 FormsModule 的时候，所有的 <form> 标签都会默认的被认为是一个 NgForm ，因此我们并不需要显式的在标签中写 ngForm 这个指令。

    #f="ngForm" 是一个 模板引用,当然，也可以这样写 ref-f="ngForm"。

    ngModel 与 name

    当在表单中使用 [(ngModel)]时，必须要定义name属性。

    在表单中的控件添加 ngModel 指令将会自动创建一个 FormControl，它们将被注入到 NgForm 指令中，而 NgFrom 指令将会被附加到 <form> 标签。而每个 FormControl 都是以 name 属性作为键值的。

    此外，通过 ngModel 指令，不仅可以跟踪控件的状态，还可以使用特定的 Angular CSS 类来更新控件，以反映当前状态，它们包括，验证的状态(valid/invalid)、控件的状态(是否获得过焦点touched/untouched，是否更改过内容pristine/dirty)。

    响应式表单

    响应式表单倾向于在非UI的数据模型（通常接收自服务器）之间显式的管理数据流， 并且用一个UI导向的表单模型来保存屏幕上HTML控件的状态和值。

    使用响应式表单需要引入一个不同的模块，它是ReactiveFormsModule。

    响应式的表单类

    1. AbstractContorl: AbstractContorl是FormControl,FormGroup,FormArray 的抽象类。为它们提供了一些共同的行为和属性，其中有些是可观察对象（Observable）。

    2. FormControl: 用于跟踪一个单独的表单控件的值和有效性状态

    3. FormGroup: 用于 跟踪一组AbstractControl的实例的值和有效性状态。组件中的顶级表单就是一个FormGroup。

    4. FormArray: 用于跟踪AbstractControl实例组成的有序数组的值和有效性状态。

    5. FormBuilder: 通过 FormBuilder 类，我们可以方便的创建出以上对象的实例，以提高我们操作复杂表单的能力。

    通常它们的关系是，如果有多个 FormControl，我们会希望把它们注册进一个父FormGroup中。FormGroup 在 HTML 模板中对应的指令是 [formGroup],而 FormControl 在模板中对应的指令是 formControl 和 formContorlName。当使用 FormGroup时，模板只能使用 formContorlName 指令。当然 FormGroup 是可以多级嵌套使用的。

    FormGroup 接受三个参数: controls一个由一系列控件(控件的名字作为key,一个key对应一个 FormControl)组成的类似字典的对象，validator（验证器） 和 asyncValidator（异步验证器） ，其中只有 controls字典对象是必须的参数，后两个都是可选参数(上面例子中没有后两个参数)。

    FormContorl

    FormControl构造函数同样接收三个参数(但是，这三个参数都是可选的)：控件初始值(formState)、控件验证器或验证器数组(validator)，异步验证器或者异步验证器数组(asyncValidator)。(上面的示例中没有提供异步验证器)

    由此可以看到，响应式表单区别于模板驱动型表单的的主要特点在于：是由组件类去创建、维护和跟踪表单的变化，而不是依赖模板。
